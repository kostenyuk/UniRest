Форма редагування запиту

Форма призначена для редагування запиту, що буде використовуватися для виконання звіту

На формі розташовується поле для редагування самого запиту, ознака необхідності автоматичного розпізнавання настройок звіту, табличне поле для редагування "твердих" параметрів запиту

Редагування запиту

Текст запиту редагується в текстовому полі. Для редагування можна користуватися конструктором запитів, що викликається за допомогою контекстного меню.

Текст запиту пишеться мовою побудовника звітів

Мова побудовника звіту

Мова побудовника звіту являє собою текст запиту з додатковими синтаксичними елементами, призначеними для роботи побудовника.
Елементи мови побудовника виділяються у фігурні дужки
Існують наступні елементи:

ВИБРАТЬ - у цій пропозиції описуються поля, які користувач зможе вибирати для виведення
ГДЕ - описуються поля, на які користувач зможе накладати обмеження
УПОРЯДОЧИТЬ ПО - описуються поля для позначення порядку
ИТОГИ ПО - описуються поля, по яких будуть виводитися підсумкові значення

Після всіх цих елементів через кому перераховуються поля, які можуть бути як "реальними" полями, так і псевдонімами полів зі списку вибірки. Після поля можна вказувати псевдонім поля для побудовникя звіту
Приклад:

{ИТОГИ ПО Номенклатура КАК Товар, Контрагент КАК Покупатель}

Якщо після поля поставити ".*", те це буде означати, що крім самого поля побудовникь дасть можливість використовувати й всі поля "через крапку" від даного поля
Приклад:

{ИТОГИ ПО Номенклатура.* КАК Товар, Контрагент КАК Покупатель}

В даному прикладі користувач зможе вибирати як вимір не тільки номенклатуру але також і всі її реквізити, наприклад Номенклатура.ОсновнойПоставщик".

Крім основних елементів, побудовник приймає елементи, записані в параметрах віртуальних таблиць. В таких випадках тип полів залежить від типу параметра, у якому розташовуються елементи

    УчетНоменклатурыОбороты.Склад КАК Склад,
    УчетНоменклатурыОбороты.КоличествоПриход КАК КоличествоПриход,
    УчетНоменклатурыОбороты.КоличествоРасход КАК КоличествоРасход
ИЗ
    РегистрНакопления.УчетНоменклатуры.Обороты({&ДатаНачала}, {&ДатаКонца}, ,{Номенклатура.*, Склад.*}) КАК УчетНоменклатурыОбороты

В цьому прикладі поля Датаначала, Датаконца, Номенклатура, Склад стануть доступними у відборі, тобто користувач зможе застосовувати для них фільтри

Можливе позначення таблиць як "необов'язкових". Це робиться за допомогою виділення частини з'єднання у фігурні дужки. У цьому випадку побудовник звіту буде включати виділене з'єднання в результуючий запит тільки тоді, коли користувач де-небудь задіє будь-яке поле з таблиць з'єднання

Автоматичне заповнення параметрів

Якщо виставити прапорець "Автоматичне заповнення параметрів" , тоді настройкизвіту будуть завантажуватися із запиту автоматично.

При автоматичному заповненні будуть виконані наступні дії:

  • Поля зі списку вибірки і всі їхні підлеглі поля добавляться в доступні поля порядку, відбору, обраних полів
  • Поля зі списку вибірки добавляться в список обраних полів
  • Поля і їхні підлеглі поля із речення "ИТОГИ ПО" добавляться в поля, доступні для вибору в виміри.
  • Поля із речення "ИТОГИ ПО" добавляться у виміри по рядках
  • Параметри віртуальних таблиць добавляться в доступні відбори

Настройки параметрів

Настройка параметрів здійснюється  в табличному полі. Кожний параметр має ім'я, значення параметра, список значень або рядок тексту, яку необхідно виконати для одержання значення параметра

Настройки представлень

На закладці "Представлення" можна настроїти представлення полів для відображення в звіті.

Виведення звіту

На закладці "Виведення звіту" можна настроїти, куди здійснюється виведення звіту, а також настроїти розшифровку звіту за замовчуванням
